Considerations

Why does your site exist?

This is the main thing to keep in mind when making a site: the purpose of its existence. Don't make a site just because you feel like you should, make a site because it's the best way to accomplish a goal that you have. That goal will also inform what requirements you have for a platform, since it doesn't matter how nice a platform's text editor is if you need to host an interactive map. You might also tolerate a longer update process for a site you update once a year, but not for one you need to keep current on a weekly basis.

Non-negotiables

  • It needs to be accessible. (talk to DAS)
  • Sites require ongoing maintenance.

Two things that aren't negotiable about any site: accessibility and maintenance. Neurodiverse people and people with different kinds of physical ability exist in the world and will be part of the audience for any site, so you want to make sure that your site isn't built to exclude them. If you're making a site and want advice on making it more accessible, talk to Digital Accessibility Services. They're great. Also, sites require ongoing maintenance. For Harvard platforms, that's done within HUIT and you don't have to pay for it as a Harvard affiliate. If you're going outside of Harvard, you'll need to provide money and/or labor to maintain your site.

Options

So, let's take a look at some Harvard-supported options for getting stuff online.

Open Scholar

  • Based in Drupal
  • Supported by Harvard Web Publishing
  • Robust SEO and accessibility testing
  • Built with academics in mind
  • Some support and customization is fee for service

Most of the department sites you'll see in the FAS run on OpenScholar. It's the main platform for scholarly portfolios, departments, and projects at Harvard. It's supported by Harvard Web Publishing (or HWP), who've tested for accessibility and make sure pages show up in search results. It can be great for portfolios and projects, but editing can be clunky and feel limiting, and some support and customization through HWP is fee for service.

Campus Press

  • Based in WordPress
  • New offering from Harvard Web Publishing
  • Robust accessibility testing
  • More familiar interface / environment
  • Limited/vetted set of plugins & themes

Campus Press is a new offering from Harvard Web Publishing, and it's a WordPress-based platform. The WordPress interface is likely to be at least a little familiar, as WordPress powers something like a third of the web. It gets the same kind of accessibility testing for supported themes as Open Scholar, but in a different environment. Campus Press only makes a certain vetted subset of themes and plugins available, so you won't have as many options as you would with self-hosted WordPress, but you'll know that the options you have are accessible and secure.

Omeka (Classic)

  • Supported by FAS Academic Technology
  • Harvard theme is tested for accessibility
  • Libraries / Archives / Museums tool
  • Metadata-focused
  • Support for IIIF images
  • Unique plugins like Neatline

Omeka is a platform that we in FAS Academic Technology provide. It's mostly used in libraries, archives, and museums, since it's very much focused on collections of stuff and the metadata that describe that stuff. It also has support for IIIF, which is a standard for displaying images online used by many libraries, archives and musuems, including Harvard's, so you have tools available in Omeka for working with those images in more robust ways, like gathering them into workspaces and annotating them. Omeka also has some plugins that we support that provide unique functionality, like the Neatline plugin for annotating maps.

Scalar

  • Supported by FAS Academic Technology
  • Accessibility improvements are ongoing
  • Built for scholarly monographs
  • Good for using multimedia, nonlinear narratives
  • Use connections between pages to create widgets
  • Can easily get a class roster imported

Scalar is another platform that we support in FAS Academic Technology. Scalar is focused more on monograph-style scholarly works, but with an emphasis on multimedia and nonlinearity. You can make a site that makes effective use of images and videos with Scalar, and provide an organization that encourages wandering and exploration over a straight linear path (although linearity is also an option). You can also use the different connections between parts of your website to represent connected pages on maps, timelines, or other visualizations. It's often used in classes for non-traditional and collaborative writing assignments, so we have tools to help import class rosters.
